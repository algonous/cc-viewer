# cc-viewer

Web-based session browser for Claude Code conversation history.

Browse, inspect, and export your Claude Code sessions with token/cost data.
Serves an embedded web UI from a single Go binary -- no Node.js, no build step.

## Install

```
make
```

Builds to `~/.local/bin/cc-viewer` by default. Override with `BIN=/your/path make`.

## Usage

```
cc-viewer [--claude-dir ~/.claude] [--addr 127.0.0.1:0]
```

Starts a local HTTP server and prints the URL. Open in a browser.

### Viewer sections

Each round displays labeled chat blocks with fold/unfold toggles:

- **YOU** (amber) -- user message, open by default
- **CONTEXT** (purple) -- system-injected context, folded by default
- **TOOL** (green) -- tool calls as structured list, folded by default
- **THINKING** (gray italic) -- chain of thought, folded by default
- **CLAUDE** (blue) -- assistant response, open by default

Content within blocks is rendered server-side via the [md2html](https://github.com/algonous/md2html) pipeline. Block structure and layout are owned by the frontend.

### Selection and Copy

Click the checkbox on any block to select it. Selected blocks show a cyan outline. The toolbar shows **Copy** (copies selected block text to clipboard) and **Clear** buttons when blocks are selected.

### Export

Click the **Export** button in the viewer toolbar. Choose format:

- **JSONL** -- structured data, one JSON line per round
- **Markdown** -- readable markdown with fenced code blocks

Exports download directly to your browser's default download folder.

## Token usage

Each round header shows compact token counts:

| Field | Meaning                                                                 |
|-------|-------------------------------------------------------------------------|
| `in`  | Input tokens sent to the model (prompt, context, tool results)          |
| `out` | Output tokens generated by the model (response text, tool calls)        |
| `cr`  | Input tokens served from prompt cache (not re-processed, ~90% cheaper)  |
| `cw`  | Input tokens written into prompt cache for the first time               |

## Data sources

Reads from `~/.claude/`:
- `history.jsonl` -- session index (grouped by sessionId)
- `projects/<encoded-dir>/<sessionId>.jsonl` -- full transcripts

## Architecture

```
main.go                  -- HTTP server entry point, go:embed web/
internal/
  data/
    types.go             -- Round, SessionSummary, Usage, export types
    history.go           -- parse history.jsonl into SessionSummary list
    transcript.go        -- parse per-session JSONL into Transcript (rounds)
    export.go            -- GenerateJSONL, GenerateMarkdown (in-memory)
  server/
    server.go            -- HTTP handlers, md2html rendering, JSON API
web/
  index.html             -- SPA skeleton
  static/
    app.css              -- theme (matches md2html color palette)
    app.js               -- all frontend logic, mouse-only interaction
```

The server sends structured JSON per round (`user_html`, `assistant_html`, `thinking_html`, `tool_calls[]`). The frontend builds block layout (folds, checkboxes, roles) from this data. md2html only renders markdown content within fields, not block structure.
