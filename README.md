# cc-tree

TUI session browser for Claude Code conversation history.

Browse, inspect, and export your Claude Code sessions with token/cost data.

## Install

```
make
```

Builds to `~/.local/bin/cc-tree` by default. Override with `BIN=/your/path make`.

## Usage

```
cc-tree [--claude-dir ~/.claude]
```

### Key bindings

| Key   | Action                     |
|-------|----------------------------|
| Up/Down | Navigate sessions        |
| Enter | Select session             |
| Tab   | Switch focus (sidebar/viewer) |
| d     | Export selected session     |
| /     | Filter sessions            |
| q     | Quit                       |

## Token usage

Each round and the session summary show token counts from the Claude API:

| Field         | Meaning                                                                 |
|---------------|-------------------------------------------------------------------------|
| `in`          | Input tokens sent to the model (prompt, context, tool results)          |
| `out`         | Output tokens generated by the model (response text, tool calls)        |
| `cache_read`  | Input tokens served from prompt cache (not re-processed, ~90% cheaper)  |
| `cache_write` | Input tokens written into prompt cache for the first time (~25% more expensive than uncached input) |

In a typical session, `cache_read` dominates because the conversation context is cached
across turns. Only new or changed parts of the prompt incur `cache_write` or uncached `in` cost.

## Data sources

Reads from `~/.claude/`:
- `history.jsonl` -- session index
- `projects/<encoded-dir>/<sessionId>.jsonl` -- full transcripts

## Export

Press `d` to export the selected session to `~/.config/cc-tree/exports/<sessionId>.jsonl`.
One line per round with user message, tool calls, assistant response, and token usage.
