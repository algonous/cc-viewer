# cc-viewer

Web-based session browser for Claude Code conversation history.

Browse, inspect, and export your Claude Code sessions with token/cost data.
Serves an embedded web UI from a single Go binary -- no Node.js, no build step.

## Install

```
make
```

Builds to `~/.local/bin/cc-viewer` by default. Override with `BIN=/your/path make`.

## Usage

```
cc-viewer [--claude-dir ~/.claude] [--addr 127.0.0.1:0]
```

Starts a local HTTP server and prints the URL. Open in a browser.

### Viewer sections

Each round displays labeled chat blocks rendered through the md2html pipeline:

- **YOU** (green) -- user message
- **CONTEXT** (dim) -- system-injected context
- **TOOL** (orange) -- tool calls (folded by default)
- **THINKING** (dim italic) -- chain of thought (hidden by default, toggle with toolbar button)
- **CLAUDE** (purple) -- assistant response

### Export

Click the Export button in the viewer toolbar. Choose format:

- **JSONL** -- structured data, one JSON line per round
- **Markdown** -- readable markdown with fenced code blocks

Check individual rounds to export only those. Export files are saved to `~/.config/cc-viewer/exports/`.

## Token usage

Each round shows token counts from the Claude API:

| Field         | Meaning                                                                 |
|---------------|-------------------------------------------------------------------------|
| `in`          | Input tokens sent to the model (prompt, context, tool results)          |
| `out`         | Output tokens generated by the model (response text, tool calls)        |
| `cache_read`  | Input tokens served from prompt cache (not re-processed, ~90% cheaper)  |
| `cache_write` | Input tokens written into prompt cache for the first time               |

## Data sources

Reads from `~/.claude/`:
- `history.jsonl` -- session index
- `projects/<encoded-dir>/<sessionId>.jsonl` -- full transcripts

## Architecture

The viewer converts each transcript round to markdown with role-tagged fenced code blocks (`you`, `tool`, `thinking`, `claude`, `context`), then renders to HTML server-side using the [md2html](../md2html) pipeline (AST -> IR -> HTML). The frontend displays the pre-rendered HTML with fold/unfold interactivity layered on via vanilla JavaScript.
